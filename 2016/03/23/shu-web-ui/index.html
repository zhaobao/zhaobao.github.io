网页中没有类似android额dp的东西，因此，如果设计图是120px，那么不论什么样的尺寸下，都是120px，这样体验是不好的，比如在一个>1000px的设备上，120px是一个不从的左侧栏宽度，但是，倒是只有320px的设备上，就显得太大了，应该是能够等比缩放比较好，比如40px；

比例的概念是不错的，问题是，网页上好多空间使用比分比是比较困难的：

  • 首先你把UI图都换算成比例
  • 页面上的元素的比例依据是父类元素，计算起来就吐血了

有没有一个不是相对父元素的东西，有，那就是rem

rem values were invented in order to sidestep the compounding problem. rem values are relative to the root html element, not the parent element. In other words, it lets you specify a font size in a relative fashion without being affected by the size of the parent, thereby eliminating compounding.

这个单位是相对html的font-size来说的，好了，实践的解决方法：根据实际的屏幕尺寸，动态计算顶级的font-size

Ratio 问题

还有一个问题，在不同的ratio下，1px的感觉是不一样的，比如，在一个ratio = 2 的分辨率下，会用两个像素点来作为1px
对图片的影响比较大，如果在一个100x100px的地方，放一个100x100px的图，在ration=1的情况下没问题，但是在ratio=2的情况下，就会显示模糊了，当然了，这个与标尺无关，需要提供额外的资源
