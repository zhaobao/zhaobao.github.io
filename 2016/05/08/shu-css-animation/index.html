CSS动画的优势

  • 易用
  • 性能高
  • 无需优化

概念

动画，动画，顾名思义，可以动的画面，CSS动画很好的诠释了这个概念，通过定义不同进度下的画面，连动起来，形成了动画
CSS通过 keyframe 来定义关键帧，通过animation-*属性来定义动画的方方面面

语法

  • 关于animation-fill-model是不好理解的，直接看下图就明白了

  • 关于animation-play-state可能会觉得莫名其妙，其实就是可以控制动画，让他暂停，然后可以继续播放，应用场景可能不多。

delay 为负数的情况

这个有点意思，可以把动画从一个中间状态开始，比如定义了一个3秒动画，delay为-1s，那初始状态就是从秒开始的状态

性能

不要以为你看懂了就很高兴了，其实还有很多，虽然好多属性可以用来动画，可以看参考链接，但是改变一个属性的成本是不一样的，可以看csstrigger来参考哪些属性会引起layout paint composition，当然了，简而言之还是要说下的：所有的动画里，posotion scale rotation 和 opacity的消耗是最低的

性能的东西不在这多说了，会单独来一个文章。

兼容性

从IE10开始，手机浏览器需要带-webkit前缀

与translate的区别？

transition定义了两种状态的转换效果，无法指定中间过程的

局限

animation的交互还是有限的，你只能去监听，却不能做到详细的控制，甚至，控制的话还不如几个transition的组合，如果动画比较复杂，建议使用canvas动画
